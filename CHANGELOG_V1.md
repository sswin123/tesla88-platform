## ERP 管理后台 (erp/)

### 核心功能

- **仪表板** — 实时数据摘要：存款、提款、会员、余额概览
- **会员管理** — 搜索、详情、余额调整、风险标记
- **存款管理** — 审核工作流（Pending → Approved / Rejected）+ 凭证查看
- **提款管理** — 审核工作流 + 处理记录
- **银行管理** — 添加/编辑/删除收款银行账户，启用/停用控制
- **促销管理** — 创建促销活动、设定金额/倍数/上限/类型（一次性/重复）
- **公告管理** — 机器人内广播公告，支持置顶消息
- **广播中心** — 向全体或筛选会员发送 Telegram 广播
- **审计日志** — 完整操作记录，不可删除
- **财务报表** — 日/周/月存提款汇总
- **会员分析** — 注册趋势、活跃度分析
- **风险中心** — 可疑行为标记与人工审查

### 控制中心

- **品牌中心 (Brand Center)** — Logo、品牌名、主色、联系方式统一管理；自动同步至机器人和网站
- **Telegram 机器人设置** — 欢迎语、客服用户名、提款限额等业务参数
- **机器人消息 CMS** — 所有机器人文案可视化编辑，无需修改代码
- **媒体库** — 集中管理图片/视频；支持 ERP 内引用

### 网站 CMS

- **网站设置** — 全局网站配置（域名、SEO、功能开关）
- **APK 管理** — 上传/管理安卓 APK，网站自动展示下载链接
- **横幅管理** — 首页轮播图上传与排序
- **网站公告** — 网站顶部滚动公告管理
- **游戏商管理** — 游戏商 Logo/链接/排序管理
- **支付银行管理** — 网站展示的充值银行列表

### 权限与安全

- **员工权限系统** — 细粒度权限引擎（20+ 权限点），角色分配
- **员工管理** — 创建/编辑员工账户与角色
- **权限管理 UI** — 可视化角色权限配置
- **JWT 中间件** — 所有 ERP 路由强制鉴权，仅公开端点豁免
- **登录限速** — 5次/15分钟/IP

### 实时客服 (Live Chat)

- **客服面板** — 实时查看/接收/处理用户客服请求
- **快速回复管理** — 分类快速回复库（支持文字、图片、视频等媒体）
- **Ctrl+V 粘贴上传** — 直接粘贴图片到快速回复编辑器
- **拖拽上传** — 拖拽文件到媒体快速回复

### 系统运维

- **健康监控 (/system/health)** — 四服务实时状态（数据库、ERP、网站、机器人），30秒自动刷新
- **备份管理 (/system/backups)** — 一键 pg_dump，自动清理超期备份（默认保留30天）
- **维护模式** — 启用/停用全站维护页面
- **系统设置** — 全局参数配置（最低存提款额、保留天数等）
- **版本显示** — 侧边栏底部显示 v1.0.0

## 会员网站 (website/)

- **暗色赌场主题** — 全站统一暗色 UI，响应式移动端适配
- **PWA 支持** — 可安装为手机应用，支持离线页面
- **会员注册/登录** — JWT 鉴权，手机号/用户名注册
- **首页内容系统** — 品牌 Logo、横幅轮播（从 ERP CMS 获取）、公告滚动
- **会员中心** — 查看余额、个人信息、修改密码
- **交易历史** — 存款/提款记录查询
- **存款流程** — 选择银行 → 填写金额 → 上传凭证 → 等待审核
- **提款流程** — 绑定银行卡 → 申请提款 → 等待审核
- **促销中心** — 查看当前活动促销
- **游戏大厅** — 展示游戏商列表（从 ERP CMS 获取）
- **在线客服** — 直接发起 Telegram 客服对话
- **网站公告** — 顶部滚动公告
- **健康检查端点** (/api/health) — 供监控系统使用

## Telegram 机器人 (bot/)

- **会员注册** — 用户名设置、银行卡绑定引导
- **主菜单** — 快捷入口：存款、提款、余额查询、促销、客服
- **存款流程** — 选银行 → 金额 → 上传凭证 → 通知财务组审核
- **提款流程** — 填写金额 → 通知财务组审核
- **余额查询** — 实时账户余额
- **促销领取** — 查看并申请促销活动
- **实时客服** — FSM 驱动的人工客服对话（用户↔客服组双向转发）
- **机器人消息 CMS** — 所有文案从数据库读取，无硬编码文字
- **品牌同步** — 品牌名/Logo 与 ERP 品牌中心同步
- **Bot Relay API** — REST API 供 ERP 触发机器人动作（发消息、撤销等）

## 基础架构

- **单根部署架构** — 一个 `.env` 驱动所有服务，`scripts/gen-env.sh` 自动生成子项目配置
- **数据库迁移系统** — 040 个迁移文件顺序执行，幂等，支持重跑
- **Docker 生产配置** — restart: always、健康检查（含 start_period）、json-file 日志轮转（10MB×5）
- **备份卷** — `erp_backups` Docker 命名卷持久化 ERP 备份
- **环境变量验证** — ERP/Website 启动时检测缺失必填项，直接 `process.exit(1)` 快速失败
- **安全头** — CSP、X-Frame-Options、X-Content-Type-Options、Referrer-Policy
- **一键更新** — `./scripts/update-system.sh`：备份→迁移→重建→健康检查
- **一键回滚** — `./scripts/rollback-db.sh`：恢复指定备份文件

## 部署说明

### 全新安装

```bash
git clone <repo-url> /opt/telegram-member-bot
cd /opt/telegram-member-bot
cp .env.example .env
# 编辑 .env 填入所有必填项
./scripts/update-system.sh
```

详见 [docs/RELEASE_V1.md](docs/RELEASE_V1.md)。

### 升级现有系统

```bash
cd /opt/telegram-member-bot
./scripts/update-system.sh --pull
```

### 环境变量必填项

| 变量 | 说明 |
|------|------|
| `BOT_TOKEN` | Telegram BotFather 颁发的 token |
| `SUPER_ADMIN_ID` | 超级管理员 Telegram 用户 ID |
| `POSTGRES_PASSWORD` | 数据库密码（强随机字符串）|
| `JWT_SECRET` | ERP session 签名密钥（64字符）|
| `MEMBER_JWT_SECRET` | 网站 session 签名密钥（64字符，不同于 JWT_SECRET）|
| `ADMIN_CHAT_ID` | 财务审核群 ID（负数）|
| `SUPPORT_CHAT_ID` | 客服群 ID（负数）|
| `BOT_RELAY_AUTH_TOKEN` | Bot Relay 鉴权 token |
